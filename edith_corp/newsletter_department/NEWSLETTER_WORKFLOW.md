# メルマガ運用ワークフロー

## 🎯 基本フロー

### 1️⃣ コンテンツ相談・作成

**鶴田さん → EDITH**
```
「今週のメルマガ作りたい」
「AI LABイベントの告知メルマガ考えて」
```

**EDITH → 鶴田さん**
- テンプレートベースの提案
- 複数案の提示
- 過去の反応データを参考にした推奨案

**鶴田さん ↔ EDITH**
- 内容の調整・修正
- 件名のブラッシュアップ
- CTA（行動喚起）の最適化

**鶴田さん決定**
```
「じゃあこれで送って」
```

---

### 2️⃣ 分散送信実行

**自動で実行される処理**:
1. 278名を5つのグループに分割（各グループ約56名）
2. 以下の時間帯に分散送信:
   - 09:00 - グループA（朝型・通勤時間）
   - 12:00 - グループB（昼休み）
   - 15:00 - グループC（午後の休憩）
   - 18:00 - グループD（帰宅時間）
   - 21:00 - グループE（夜・リラックスタイム）

3. 各グループはランダム割り当て（毎回違う人が違う時間帯に）

---

### 3️⃣ 効果測定・分析

**翌日レポート**:
- 時間帯別開封率
- 時間帯別クリック率
- 最も反応が良かった時間帯
- 配信停止数

**蓄積データ**:
- 過去の配信パターン
- 時間帯別パフォーマンス推移
- ベストプラクティスの発見

---

## 📝 テンプレート構造

### 固定部分（変更不要）
- ヘッダーデザイン
- フッター（会社情報・配信停止）
- 基本レイアウト

### 可変部分（毎回変更）
- メインコンテンツ（イベント告知、新サービス、ブログ紹介等）
- 件名
- CTA（ボタンテキスト・リンク先）

### パーソナライズ部分（自動挿入）
- `{{ contact.LASTNAME }}` `{{ contact.FIRSTNAME }}`
- グループタグ（時間帯識別用）

## 🔧 技術的な実装

### ファイル構成
```
newsletter_department/
├── templates/
│   ├── base_template.html       # 基本テンプレート
│   ├── ailab_event.html         # AI LABイベント用
│   ├── new_service.html         # 新サービス告知用
│   └── monthly_report.html      # 月次レポート用
├── tools/
│   ├── distributed_sender.py    # 分散送信マネージャー
│   ├── send_newsletter.py       # 即時送信スクリプト
│   ├── brevo_api.py             # Brevo API ラッパー
│   └── schedules/               # 送信スケジュール保存先
└── NEWSLETTER_WORKFLOW.md       # このファイル
```

### コマンド例

#### スケジュール作成（1日分散送信）
```bash
cd newsletter_department/tools
python3 distributed_sender.py create "2月AI LAB告知"
# → schedules/2月AI LAB告知_20260212_143000.json が作成される
```

#### 時間帯別送信（自動化推奨）
```bash
# 09:00 グループ送信
python3 distributed_sender.py send schedules/xxx.json 0

# 12:00 グループ送信（cronで自動実行）
python3 distributed_sender.py send schedules/xxx.json 1

# ... 以降同様
```

#### 進捗確認
```bash
python3 distributed_sender.py status schedules/xxx.json
```

## 📊 期待される効果

### A/Bテスト（時間帯別）
- どの時間帯が最も開封率が高いか
- 曜日×時間帯の組み合わせ最適化
- ターゲット層の行動パターン把握

### リスク分散
- 一斉送信でサーバー負荷集中を回避
- 受信側のスパム判定リスク低減
- 送信エラー時の影響最小化

### パーソナライゼーション
- 将来的には「開封しやすい時間帯」を学習
- 各受信者に最適な時間帯で送信
- セグメント別の配信戦略
